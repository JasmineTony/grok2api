# 第 07 次迭代计划：依赖升级与可靠性平台基础

- 日期：2026-07-20
- 负责人：JasmineTony
- 状态：实施中
- 基线提交：`46ca1f4b8b77d5d5a10a8c0059c49062a3c6e1ad`
- 工作分支：`codex/reliability-foundation`

## 目标

在保持公开 API、配置语义、数据库兼容和现有分层不变的前提下，完成第一阶段基础建设：安全依赖升级、前端性能/版式基线、统一错误与账号状态机、协议黄金文件测试和 Prometheus 基础。

## 本轮范围

1. 升级兼容的 Node.js、pnpm、Vite、Tailwind、ESLint、React 相关依赖和少量 Go 直接依赖。
2. 建立前端包体预算、路由错误边界、图表异步加载、响应式表格/Dialog 规则和基础 UI 回归测试入口。
3. 将现有上游失败分类扩展为统一稳定错误模型。
4. 增加可回滚的账号运行状态字段与状态事件表；超时、网络、代理、5xx 和未知 403 不得误触发重认证。
5. 建立离线协议黄金文件规范化测试。
6. 增加默认关闭、低基数、仅本地监听的 Prometheus 指标基础。

## 明确不在本轮

- 不实现账号级代理策略、用量汇总、通知、备份、请求策略、回放、CLI 子命令或 MCP；这些进入后续独立迭代。
- 不直接升级 TypeScript 7、Recharts 3 或跨度较大的 `modernc/sqlite` 间接依赖；先建立兼容性验证和独立迁移边界。
- 不进行全站视觉重做，不修改 Go module 路径、公开 `/v1/*` 响应、数据库既有字段含义或发布工作流。

## 兼容与安全约束

- 数据库只允许增量、可空字段和新表；迁移必须覆盖 SQLite、PostgreSQL、旧库升级和重复启动。
- 账号状态只能由显式状态转换服务写入；保留旧 `Enabled`、`AuthStatus`、失败次数和冷却字段。
- 高风险指标监听默认关闭，开启时默认绑定 `127.0.0.1`；指标标签不得包含账号 ID、请求 ID、Token、Cookie 或完整敏感模型信息。
- 黄金文件默认禁止更新，仅 `UPDATE_GOLDEN=1` 允许更新，并且不得联网。
- 不在计划、日志或测试夹具中写入真实凭据、密钥、Cookie、请求正文或私有数据。

## 实施顺序

1. 记录依赖和前端构建性能基线，创建测试/基准入口。
2. 升级兼容依赖并验证锁文件、Go 构建和 Docker 构建。
3. 完成前端分块、版式、错误边界和可访问性修正。
4. 完成统一错误模型、账号状态机和旧字段兼容映射。
5. 完成协议黄金文件与 Prometheus 基础。
6. 运行全部检查，补齐 `RESULT.md`，通过独立 PR 合并。
