## 目标

在当前第一阶段基础上完成既定的可靠性与平台路线图，使 grok2api 达到“账号池稳定、故障可解释、资源可治理、能够放心部署”的目标状态。

## 交付策略

1. 本计划使用一个长期本地分支持续实现，不因单个文件、子模块或局部测试通过而推送。
2. 可以创建本地检查点提交，但不得在本计划所有范围、测试与验收、假设与默认值核对完成前推送任何当前计划分支。
3. 仅在最终验收全部通过后执行一次统一推送，创建一个最终 PR；检查失败继续在本地修复，不创建中间 PR。
4. 紧急安全事件、需要外部协作验证或与本计划无关的独立修复，必须另建计划并得到明确确认后才可例外推送。

## 范围

### A. 第一阶段剩余验收

- 将账号状态事件接入健康写入路径，避免业务代码分散修改状态字段。
- 将 Prometheus registry 接入请求、错误、重试、账号状态、Egress、Token 与成本业务路径。
- 扩展协议黄金测试到真实 OpenAI Responses、Chat Completions、Anthropic Messages、SSE、工具调用、usage、错误、图片、视频与畸形响应。
- 完成 Playwright 三视口、亮暗主题、表格、Dialog、导航、加载/空/错状态回归。
- 继续优化入口和 Recharts 分块，完成 Recharts 3 独立兼容迁移；TypeScript 7 仅在工具链正式兼容后进入生产。

### B. 账号级代理、用量与成本治理

- 账号覆盖、Provider 节点池、全局直连回退和批量策略。
- Egress 主动健康检查、熔断、恢复探测和历史。
- 统一 Token、缓存命中率、实际/估算成本语义。
- 小时/日汇总表、后台聚合任务和多维管理视图。

### C. 版本、备份、安全升级与通知中心

- 当前仓库版本检查、上游同步提示和升级前检查。
- SQLite/媒体一致性备份；PostgreSQL/Redis 验证、清单和外部钩子。
- 备份 manifest、校验和、加密状态与恢复兼容检查。
- 站内通知和签名 Webhook，支持去重、冷却、确认、已读和保留期限。

### D. 请求策略、回放与协议转换

- 有序类型化请求规则、dry-run、命中统计和默认允许。
- 脱敏、压缩、加密、限额和 TTL 的请求快照。
- OpenAI Responses、Chat Completions、Anthropic Messages 双向规范化预览。
- 回放默认 dry-run，实际发送需管理员确认并重新经过策略。

### E. CLI、配置即代码与运维 MCP

- 增加 serve、version、doctor、config、backup、egress 子命令并保持无子命令启动兼容。
- 声明式 YAML plan/apply/export，秘密仅引用环境变量或外部 Secret。
- 本地 stdio、默认只读的运维 MCP；禁止暴露凭据、完整正文和客户端密钥。

## 兼容与安全约束

- 保持现有 `/v1/*`、配置语义、数据库已有字段、Go module 路径和发布策略兼容。
- 数据库只增加表、索引和可空/有安全默认值字段，并覆盖 SQLite/PostgreSQL、旧库升级和幂等迁移。
- 高风险能力默认关闭：外部 Prometheus、快照、实际回放、策略阻断、Webhook、MCP 写工具。
- 不在仓库、日志、通知、指标、回放或 MCP 中泄露账号、Token、Cookie、代理密码和请求敏感正文。

## 测试与验收

- 依赖：冻结锁文件安装、lint、typecheck、测试、build、审计、Go test/vet/race、govulncheck、Swagger 无漂移、多架构 Docker。
- 状态机：完整转换矩阵；超时、DNS、代理、5xx、未知 403 不触发重认证。
- 协议：JSON/SSE、工具调用、usage、错误、媒体和转换往返黄金文件稳定。
- 数据库：SQLite/PostgreSQL 新建、旧库升级、重复迁移和兼容读取。
- 性能：Selector 不回退超过 10%，指标开销不超过 2%，Dashboard 查询至少下降 50%，前端满足最终包体预算。
- 页面：375×812、768×1024、1440×900，亮暗主题和关键交互截图回归。
- 安全：指标、通知、回放、MCP 和日志敏感信息扫描通过。
- 发布：无版本标签、无 GHCR 发布；最终 PR 检查全部成功后才 squash 合并。

## 假设与默认值验收门

- TypeScript 7 在工具链正式兼容前不进入生产构建。
- Recharts 3 作为兼容迁移完成视觉与交互验证后方可采用。
- MCP 首版仅本地 stdio、默认只读。
- 默认允许策略、默认关闭回放/阻断/通知外发/外部指标。
- 任一必须破坏兼容性的变更停止实施并记录为 vNext，不得混入最终 PR。
